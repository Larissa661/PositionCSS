Position static

No CSS, position: static é o valor padrão aplicado a todos os elementos, significando que eles são posicionados conforme o fluxo normal da página, sem deslocamentos manuais. Quando um elemento está com posição estática, propriedades como top, right, bottom e left não têm efeito, pois ele não é removido do fluxo comum. Isso garante que o layout siga a ordem natural do HTML, sem sobreposição ou movimentação especial, sendo ideal para a maioria dos elementos que não precisam de posicionamento específico.

Position: relative

No CSS, position: relative permite que um elemento seja posicionado em relação à sua posição original no fluxo da página. Quando aplicado, o elemento permanece ocupando o mesmo espaço no layout, mas pode ser deslocado usando as propriedades , right, bottom e left, movendo-se visualmente sem afetar a posição dos outros elementos. Isso torna o relative útil para ajustes finos ou como referência para elementos posicionados de forma absoluta dentro dele, já que um elemento com position: absolute pode usar o elemento relativo como ponto de referência para seu próprio posicionamento.

Position: absolute

No CSS, position: absolute remove o elemento do fluxo normal da página e o posiciona em relação ao seu ancestral mais próximo que tenha um posicionamento diferente de static (como relative, absolute, fixed ou sticky). Isso permite colocar o elemento exatamente onde se deseja usando top, right, bottom e left, sem afetar o layout dos demais elementos ao redor. Por estar fora do fluxo, outros elementos se comportam como se ele não existisse, tornando o absolute ideal para criar sobreposições, pop-ups, ícones posicionados sobre imagens e componentes que exigem um controle mais preciso da posição.

Position: fixed

No CSS, position: fixed fixa o elemento em relação à janela de visualização (viewport), mantendo-o sempre no mesmo lugar, mesmo quando o usuário rola a página. Elementos com fixed são removidos do fluxo normal do documento e podem ser posicionados com top, right, bottom e left, permitindo criar cabeçalhos fixos, botões flutuantes, barras laterais e outros componentes que devem permanecer visíveis o tempo todo. Como o posicionamento é relativo ao viewport, o elemento não se move junto com o conteúdo, garantindo sua permanência na tela independentemente da rolagem.
